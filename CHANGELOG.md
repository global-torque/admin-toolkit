# Changelog

## 0.2.0-beta.3 - 2026-07-13

- Prepared the independently reviewed admin contracts and runtime helpers for
  protected public `main` with SHA-pinned CI, public API governance,
  clean-source artifact manifests, and provenance workflow.
- Switched development and release verification to the exact published
  `@global-torque/design-tokens@0.1.0-beta.3` registry version.
- Superseded the beta.2 dependency candidate and dirty-tree beta.1
  implementation artifact.

## Pre-0.2 default-branch quarantine - Superseded 2026-07-13

- Quarantined the unsupported pre-0.2 default-branch bridge, removed install
  guidance, and disabled publication pending exact upstream assets and real
  consumer evidence. The reviewed beta.3 source superseded that quarantine
  after its exact design-token dependency became publicly available.

## 0.2.0-beta.2 - Superseded local candidate

- Superseded before publication because its exact design-token dependency was
  never released.

## 0.2.0-beta.1 - 2026-07-11

- Froze publication after the 0.2 audit invalidated the earlier readiness
  assessment.
- Replaced prose metadata with a frozen `0.2.0` schema containing structured
  elements, attributes, regions, states, tokens, keyboard/focus rules, behavior
  IDs, and complete HTML/Django fixtures.
- Added safe find/get lookup with `UnknownAdminPatternError`, per-instance and
  per-prefix Alpine registration, unique toast IDs, tabs, and accessible dialog
  focus/Escape/return-focus cleanup.
- Added htmx 2 `onLoad`/`off` installation, repeat-safe/forced Alpine hydration,
  neutral `gt-*` reference styles backed by the design-token peer, axe/DOM/real
  Alpine/htmx tests, API reports, and clean-room packaging gates.
- Replaced the private sibling design-token build fallback with an exact
  `0.1.0-beta.1` development dependency and isolated reviewed-artifact
  verification.
- Added hash-locked Django 5.2.16 source verification that renders every
  catalog fixture from committed contexts and compares exact escaped outputs.
- Added a real Chromium regression gate for selected/all-disabled tabs,
  re-enabling, non-tab focus ownership, same-task disable/re-enable, and
  explicit-blur behavior.
- Hardened nested/topmost dialog ownership, capture-phase Escape and close
  handling, inert background stacks, exact focus restoration, reentrant
  lifecycle cleanup, and single-live-controller enforcement.

## 0.1.0

- Consolidated prepare-next typed HTML pattern contracts and Alpine/htmx helper
  behavior into `@global-torque/admin-toolkit`.
- Added behavior tests for unique ids, required implementation metadata,
